Goal Considerations

  1. Define a goal. 2. Figure out how to measure that goal!

In the real world people won't tell you what algorithm to use or even what type of ML (e.g., supervised vs. unsupervised) to use! How to model your problem with ML tools? Should you use ML at all?

Thinking in Context!

What is the baseline?

What is the benefit?

What do you want to achieve? What is the easiest way to achieve this? And what will improving over this baseline buy you?

Electronics store example: closing down, want to send email to customers for final sale. Use ML to predict which customers might respond. But why? It's a closing sale.

Good and Bad Substitutes

Sometimes it can be hard to find the right metric. It needs to measure what you actually care about, or something close to it, and you actually have to be able to measure it. Often you need to find a good substitute metric for what you really want b/c what you really want is too hard to measure.

Ex: ad click rates. Doesn't necessarily translate to sales. Easy to influence by putting ads next to what users often click on, or moving stuff around after loading, or other dark patterns.

Communicating Results

Explain to others, e.g., your boss, why it works, nothing weird will happen. An important skill.

Explainable Results

Recommendation engines work better if you can explain to users why you made those choices. Sometimes there's a balance between this and good prediction accuracy.

Ethical Considerations

Sometimes explanations and transparency are REALLY important, e.g. when lives are at stake. Ex: risk assessment system that had racial bias. Situation might have been better if the system was more transparent.

About Python

Why Python

  • General purpose language
  • Great libraries
  • easy to learn / use
  • Alternatives: R, Julia, Matlab

Unlike Matlab and R, python is general purpose. Lots of libraries since it's so popular, so most things you can do in a few lines. But R is a really good option, and from the statistical side it's actually more popular.

Why Not Python

  • Not made for statistical computing
  • Slow!
    • But many libraries are written in C. Use them!
  • Two major, incompatible versions of Python
    • 2 vs. 3 (we'll use 3)

Well, it's general purpose…but this isn't too much of a downside really. But Python really is pretty slow. For small programs you don't notice this much, but when you're doing data analysis you definitely do. It's also not very good at parallelism.

However, Python let's you write some code in C and use it from Python. So in fact lots of the libraries we're going to use have a Python interface but are written in C, so they are pretty fast. So if you wanted to implement your own algorithm and couldn't do it well by using the libraries we have, you'd have to write in C and figure out how to interface with Python. But you won't need to do that in this class.

Also, two versions of Python. print, division, etc.

Python Packages

  • If you have multiple versions of Python…it can be a mess
    • Look up "virtual environments"
  • To install python packages (libraries), can use "pip"
  • But easier to use anaconda
    • Comes with the packages we will want to use

Dynamically typed and interpreted

  • Less to type, not int, double, etc.
  • But you lose the safety net of types
    • Errors shift from compilation to runtime errors
    • Need more documentation

Actually later versions of Python3 include type annotation, but they're not supported everywhere yet.
